

# Developer Portals

## Overview

Developer portals provide a centralized entry point where engineering teams can discover platform capabilities, access self-service workflows, find documentation, manage services, and interact with internal developer platforms.

Platform engineering teams use developer portals to reduce cognitive load, improve developer experience, increase platform adoption, and provide a consistent interface to engineering systems.

Developer portals are a foundational capability within platform engineering, internal developer platforms, developer experience programs, and large-scale engineering organizations.

- Better Developer Experience

- Reduced Cognitive Load

## Core Portal Capabilities

### Service Catalog

Provides visibility into engineering assets.

Examples:

- Applications
- Services
- APIs
- Infrastructure Components

---

### Documentation Hub

Centralizes engineering knowledge.

Examples:

- Platform Documentation
- Runbooks
- Architecture Guides
- Operational Procedures

---

### Self-Service Workflows

Provides platform capabilities through automation.

Examples:

- Environment Provisioning
- Deployment Requests
- Resource Creation
- Access Requests

---

### Platform Visibility

Provides insight into platform and service ownership.

Examples:

- Ownership Information
- Dependency Mapping
- Operational Status
- Service Health

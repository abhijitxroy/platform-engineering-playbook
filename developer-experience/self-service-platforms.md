

# Self-Service Platforms

## Overview

Self-Service Platforms provide developers with the ability to access infrastructure, platform capabilities, environments, deployment workflows, and engineering services without requiring manual intervention from platform or operations teams.

Platform engineering teams build self-service platforms to reduce bottlenecks, improve developer productivity, accelerate software delivery, and increase platform adoption.

Self-service platforms are a foundational capability within platform engineering, internal developer platforms, developer experience programs, and modern engineering organizations.

---

## Why Self-Service Platforms Matter

Without Self-Service Platforms:

```text
Developer Request
        ↓
Manual Approval
        ↓
Platform Team Dependency
        ↓
Slow Delivery
```

With Self-Service Platforms:

```text
Developer Request
        ↓
Platform Automation
        ↓
Immediate Access
        ↓
Faster Delivery
```

Benefits:

- Faster Software Delivery
- Reduced Operational Bottlenecks
- Improved Developer Experience
- Increased Productivity
- Better Platform Adoption

## Core Self-Service Capabilities

### Environment Provisioning

Developers can create environments without manual support.

Examples:

- Development Environments
- Testing Environments
- Sandbox Environments

---

### Deployment Enablement

Developers can deploy applications through platform workflows.

Examples:

- CI/CD Integration
- Deployment Automation
- Release Workflows

---

### Resource Requests

Developers can access platform resources through automation.

Examples:

- Databases
- Storage
- Messaging Systems
- Compute Resources

---

### Platform Automation

Automates common operational activities.

Examples:

- Infrastructure Provisioning
- Access Management
- Configuration Management

## Production Challenges

Common challenges include:

- Governance Requirements
- Security Controls
- Resource Sprawl
- Poor User Experience
- Insufficient Automation
- Platform Adoption Challenges

Successful self-service platforms balance autonomy, governance, security, and usability.
